# ODD+D Model Description

## I. Overview

### I.1 Purpose

The model investigates whether an audited, temporally ordered information environment can produce emergent changes in synthetic voting states through selective exposure, adaptive trust, finite motif memory, semantic reframing, multiplex social transmission, bounded-confidence interaction, and post-ABM LLM deliberation.

The intended audience is researchers and advanced students in agent-based modelling, computational social science, information dynamics, natural-language systems, and multi-agent systems.

### I.2 Entities, state variables, and scales

#### Entities

- synthetic voter agents;
- source observations and canonical information signals;
- event clusters;
- source categories and named sources;
- semantic motifs, frames, emotions and campaign terms;
- multiplex network ties;
- exposure, transmission and interaction events;
- cognitive dossiers and structured LLM decisions.

#### Agent states

Principal endogenous states include ideological/decision position, current synthetic voting state, confidence, uncertainty, fatigue, mobilisation, blank-vote propensity, abstention propensity, source trust, motif memory, semantic exposure, accumulated information dose, and network context.

#### Exogenous inputs

Exogenous drivers are the audited source observations, their dates, event clusters, source attributes, candidate scope, direction, intensity, credibility, misinformation risk, probable channels, relevant profiles, motifs, frames, emotions and text summaries.

#### Scales

- Time: one day.
- Horizon: twenty days.
- Population: 30, 200, or 1,000 agents by profile; 1,000 in the full run.
- Geography: synthetic heterogeneity attribute, not a spatial prediction grid.
- Networks: four multiplex layers.

### I.3 Process overview and scheduling

For each simulated day:

1. determine eligible signals from the date and active-window contract;
2. calculate or retrieve daily information-environment metrics;
3. sample direct exposures according to visibility, relevance, access and model weight;
4. interpret each signal using agent state, source trust and narrative properties;
5. update position, confidence, uncertainty and related propensities;
6. update adaptive source trust and motif memory;
7. initiate bounded cascades through eligible network layers;
8. sample pairwise/local social interactions;
9. apply end-of-day fatigue, mobilisation and state classification;
10. record agent-level events and macro-level emergence metrics.

After day twenty, cognitive dossiers are produced. The LLM stage returns a second synthetic decision for comparison but does not alter the completed ABM history.

## II. Design concepts

### II.1 Theoretical and empirical basis

The model combines bounded rationality, selective exposure, adaptive trust, memory decay, bounded confidence, multiplex networks, information entropy, narrative diffusion, semantic context and constrained LLM deliberation.

The empirical information input is audited and source-category balanced. The population and networks are synthetic.

### II.2 Individual decision-making

Heuristic agents do not solve a complete utility-maximisation problem. Their decisions are path-dependent state classifications generated by bounded update rules. The effect of an exposure depends on prior state, candidate scope, direction, source trust, credibility, misinformation risk, profile relevance, emotion, motif memory and social context.

The LLM agent performs a dossier-constrained interpretation. It chooses among the allowed synthetic states and provides structured reasoning. This is a model-generated decision, not an observed person.

### II.3 Adaptation and learning

The ABM includes adaptation because trust, memory, confidence, uncertainty, fatigue, mobilisation, blank-vote propensity and abstention propensity change over time. Behavioural equations remain fixed during a run; agents do not learn new update rules.

The LLM is not fine-tuned or retrained. Its adaptation is contextual within the dossier only.

### II.4 Sensing

Agents do not observe all available signals. They receive stochastic, filtered subsets based on temporal eligibility, channel access, source visibility, relevance and network transmission. Sensing is therefore partial and heterogeneous.

### II.5 Prediction

Heuristic agents do not explicitly forecast future events. Their current state summarises accumulated exposure and interaction. The LLM reasons from the dossier and should not introduce external future information.

### II.6 Interaction

Interaction occurs through:

- direct source exposure;
- peer transmission of signal/motif content;
- bounded cascade propagation;
- local bounded-confidence interaction across four network layers.

Effects attenuate with transmission depth and are recorded for attribution.

### II.7 Collectives

Collective patterns include state shares, entropy, polarisation, cascade concentration, community agreement, semantic ownership, source impact, regional/profile differences and ABM–LLM transition structure. These are measured outcomes, not imposed targets.

### II.8 Heterogeneity

Agents differ in synthetic geography, generation, socioeconomic and occupational attributes, media access, digital exposure, ideological orientation, trust, susceptibility, confidence, uncertainty, mobilisation, network position and language-style controls.

### II.9 Stochasticity

Randomness enters population generation, network construction, signal sampling, agent ordering, cascade selection and interaction sampling. Python and NumPy are seeded. The remote LLM layer may remain non-deterministic.

### II.10 Observation and emergence

The model records exposures, transmissions, interactions, state changes, trust, memory, semantic dose, cascades, daily metrics and final decisions. A macro-pattern is considered emergent when it is produced by micro-level updates and interactions rather than assigned directly.

## III. Details

### III.1 Implementation

The model is implemented in Python in a Jupyter/Google Colab notebook. Principal libraries include pandas, NumPy, NetworkX, matplotlib, Plotly, Pydantic and the Anthropic Python SDK. Outputs are exported as CSV, JSON/JSONL, GraphML, PNG, HTML, Markdown and ZIP artifacts where applicable.

### III.2 Initialisation

The runtime establishes a temporal contract, run profile and seed. The audited ZIP is safely extracted and validated. Synthetic agents and multiplex networks are generated from configured distributions. Initial behavioural states and trust structures are assigned before day one.

### III.3 Input data

The v5.1 package contains 207 master observations, 201 canonical signals, six supporting observations, 201 event clusters and nine source categories. The model uses source-category and event-cluster weights for experimental sampling.

### III.4 Submodels

#### Data ingestion and provenance

Validates files, dates, schemas, IDs, cluster integrity, weights and checksums.

#### Daily information environment

Activates signals by date/window and computes information entropy and pressure indicators.

#### Synthetic population

Creates heterogeneous agent attributes under configured regional and profile distributions.

#### Multiplex network construction

Builds and validates four synthetic social layers.

#### Direct exposure

Samples a bounded number of eligible signals conditioned on access, relevance and weights.

#### Interpretation

Transforms exposure into bounded state changes conditioned on trust, credibility, risk, direction, motif and prior state.

#### Trust and memory

Adapts source trust and applies finite, decaying motif memory.

#### Social diffusion

Transmits selected narrative content through networks with bounded depth, fan-out and decay.

#### End-of-day adaptation

Updates fatigue, mobilisation, uncertainty and behavioural propensities, then classifies the current voting state.

#### Semantic reframing

Extracts and aggregates terms, context, emotion and semantic-dose relationships across exposures and outcomes.

#### Cognitive dossiers

Summarises the relevant agent history for a constrained LLM decision.

#### LLM deliberation

Submits structured requests, validates responses, maps anonymous options, and compares final LLM states with heuristic ABM states.

#### Reporting and dashboard

Exports analytical tables, figures, agent records, checkpoint manifests and a self-contained HTML dashboard.

## IV. Decision-specific ODD+D extensions

### IV.1 Decision subjects

Decision subjects are synthetic voter agents. The modeller defines the option set and update rules; no human participant is making decisions inside the experiment.

### IV.2 Decision objects

The primary decision object is the synthetic final state among Cepeda, De la Espriella, Voto en blanco, No alineado and Abstencion.

### IV.3 Objectives

Agents do not optimise a single explicit objective. Their state reflects consistency with orientation, trusted information, uncertainty, mobilisation, fatigue and accumulated narrative exposure.

### IV.4 Decision process

The heuristic process is incremental and path-dependent. The LLM process is a one-shot structured deliberation over the final dossier unless a separate multi-turn protocol is explicitly implemented.

### IV.5 Information and uncertainty

Agents have partial, filtered information. Uncertainty is an endogenous state. Missing engagement metrics are not interpreted as zero exposure.

### IV.6 Institutions and norms

Institutional trust, source categories, legal/status labels, blank-vote rules and campaign context enter as modelled attributes. They do not constitute legal advice or definitive adjudication of disputed claims.

### IV.7 Adaptation feedback

Source trust and motif memory create feedback from past exposure to future interpretation. The LLM result does not feed back into the ABM in version 1.0.0.

## V. Verification and validation status

The release contains automated checks for package counts, cluster integrity, result-table contracts, notebook structure and publication hygiene. Multi-seed uncertainty quantification, independent replication and external predictive validation remain future work.
