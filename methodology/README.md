# Methodology Documentation Index

This directory separates the scientific specification from the implementation notebook.

| Document | Purpose |
|---|---|
| `ODD_D_MODEL_DESCRIPTION.md` | Full Overview, Design Concepts, and Details specification with decision extensions. |
| `MODEL_CARD.md` | Intended uses, non-uses, architecture, outputs, risks, and maintenance status. |
| `DATA_CARD.md` | Input-package provenance, composition, schema, gaps, and rights boundary. |
| `DATA_DICTIONARY.md` | Field-level definitions for the master signal table and principal result tables. |
| `SOURCE_BALANCING.md` | Source-category weights, event-cluster logic, and interpretation restrictions. |
| `ABM_RULES.md` | Scheduling, state variables, exposure, memory, trust, network and transition mechanisms. |
| `LLM_DELIBERATION_PROTOCOL.md` | Cognitive dossiers, provider configuration, structured output, validation and failure handling. |
| `VALIDATION_PROTOCOL.md` | Structural, behavioural, output, linguistic and reproducibility checks. |
| `REPRODUCIBILITY.md` | Environment, seeds, manifests, checksums, profiles and release procedure. |
| `ASSUMPTIONS_AND_LIMITATIONS.md` | Explicit modelling assumptions and their consequences. |
| `ETHICS_AND_LIMITATIONS.md` | Political-use boundary, synthetic-population limits, bias risks and prohibited interpretations. |
| `GLOSSARY.md` | Controlled terminology used across the repository. |
| `REFERENCES.md` | Methodological and implementation references. |

The notebook remains the executable specification. These documents describe the public `1.0.0` release and must be revised whenever data, rules, prompts, schemas, or model versions change.
